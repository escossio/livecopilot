# Multi-domain Execution Summary Template

Preencha este documento ao final de cada rodada que processar múltiplos domínios. Use as seções abaixo para cobrir os elementos mínimos exigidos pelo runbook.

## DOMÍNIOS PROCESSADOS
Liste os domínios incluídos nesta rodada, indicando o status (por exemplo, concluído, em andamento, divergência registrada).

## ETAPAS ALCANÇADAS POR DOMÍNIO
Para cada domínio listado acima, descreva até qual etapa do pipeline chegou e se houve interrupções. Mencione quaisquer etapas pendentes ou reexecutadas.

## ARTEFATOS GERADOS
Detalhe arquivos, relatórios ou documentos novos produzidos durante a rodada, incluindo arquivos de lock, manifestos, chunk reports ou handoffs relevantes.

## DIVERGÊNCIAS IDENTIFICADAS
Registre aqui quaisquer divergências detectadas (use o formato descrito no runbook) e o impacto no cronograma.

## AÇÕES RECOMENDADAS
Liste próximos passos claros, quem deve conduzi-los (quando aplicável) e prazos estimados, se houver.

## STATUS FINAL DO LOTE
Indique o status global da rodada (ex.: `concluído`, `em andamento`, `aguardando aprovação externa`) e confirme que o resumo foi anexado ao handoff apropriado.
